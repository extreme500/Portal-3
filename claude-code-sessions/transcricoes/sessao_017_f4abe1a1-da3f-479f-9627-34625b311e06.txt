## Design do módulo `collisions`

**`include/collisions.h`** — tipos, constantes e declarações:

```cpp
// Tipos geométricos (movidos do main)
struct CollisionAABB  { glm::vec3 min, max; };
struct CollisionLine  { glm::vec2 p1, p2; float y_min, y_max; };
struct PlayerCollider { float raio = 0.25f; float altura = 1.4f; };

// Estado ESTÁTICO do cenário, construído uma vez
struct CollisionWorld {
    std::vector<CollisionAABB> aabbs;
    std::vector<CollisionLine> lines;
    CollisionAABB closedDoor;
    PlayerCollider player;
    glm::vec3 boxLocalMin,   boxLocalMax;    // bbox local do "Cube"  (cacheada)
    glm::vec3 radioLocalMin, radioLocalMax;  // bbox local do "Shell" (cacheada)
};

// Estado DINÂMICO de um frame (montado a partir das globais do main)
struct SceneState {
    glm::vec3 boxPos;   float boxAngleY;
    glm::vec3 radioPos; float radioAngleY;
    bool holdingBox, holdingRadio, buttonPressed;
};

// Geometria local lida do g_VirtualScene, passada na construção (desacopla de SceneObject)
struct CollisionSceneGeometry {
    glm::vec3 buttonMin, buttonMax,  doorMin, doorMax;
    glm::vec3 boxMin, boxMax,        radioMin, radioMax;
};

constexpr float COLLISION_SKIN       = 0.02f;
constexpr float PLAYER_FEET_Y_OFFSET = -1.0f;
```

**Assinaturas** (estado sempre passado explicitamente, zero `extern`):
- Puras: `CylinderIntersectsAABB(...)`, `CylinderIntersectsLine(...)`, `ComputeWorldAABB(...)`
- Queries: `PlayerCollidesAt(world, state, pos)`, `BoxCollidesAt(world, state, pos)`, `RadioCollidesAt(world, state, pos)`, `IsButtonTriggered(state, playerPos)`, `IsPlayerOnGround(world, state, playerPos)`
- Movimento (referências mutáveis em vez de globais): `TryMovePlayer(world, state, playerPos&, dx, dz)`, `TryMovePlayerVertical(world, state, playerPos&, velY&, dy)`
- Construção: `CollisionWorld BuildCollisionWorld(const CollisionSceneGeometry& geo)` (ex-`SetupCollisionAABBs`, retorna por valor)

**Chave do desacoplamento:** `collisions.cpp` nunca enxerga `SceneObject` nem `g_VirtualScene`. As bboxes locais do Cube/Shell/botão/porta são lidas no `main.cpp` e injetadas via `CollisionSceneGeometry` / cacheadas no `CollisionWorld`.

No `main.cpp`: `g_CollisionAABBs`/`g_CollisionLines`/`g_ClosedDoorAABB`/`g_PlayerCollider` → um único `CollisionWorld g_CollisionWorld;`, e um helper `SceneState CurrentSceneState()` que monta o estado dinâmico nos call sites.
