Boa pergunta — o conserto que fiz resolve o problema *agora*, mas é frágil: cada nova textura usada em um branch novo do shader pode estourar o limite de 16 de novo, e aí vamos ter que ficar "caçando" samplers pra remover.

Para não cair nessa armadilha de novo, as opções sustentáveis são:

1. **`sampler2DArray` (texture arrays)** — agrupar texturas do mesmo "grupo" (ex: todas as texturas do cubo, ou do Chell) em uma única array de textura, acessada por **um único sampler** + um índice de camada (`uniform int`). Isso é o que reduz o número de samplers de verdade, independente de quantas texturas você tenha. É a solução mais robusta, mas exige que as texturas do grupo tenham o mesmo tamanho (dá pra redimensionar no carregamento).

2. **Atlas de texturas** — combinar várias texturas pequenas em uma imagem só e usar sub-regiões via offset/escala de UV. Funciona bem pra texturas pequenas (botões, ícones), mas é chato de manter manualmente.
