Eram **três** problemas distintos no macOS (todos passavam despercebidos no Linux/Windows do lab):

**1. Não compilava** — o projeto passou a usar a SoLoud, mas os Makefiles do macOS não a incluíam.
- `Makefile.macOS` e a branch Darwin do `Makefile`: adicionei os fontes da SoLoud (via `wildcard`, espelhando o `CMakeLists.txt`), o include `soloud/include`, `-DWITH_MINIAUDIO` e os frameworks.

**2. Tela preta ao clicar em Jogar** — o fragment shader declarava **32 samplers** (`TextureImage0..31`), mas o macOS limita o fragment shader a **16**. O programa de GPU falhava no *link* → nada renderizava (só o texto, que usa outro programa).
- `shader_fragment.glsl`: reduzido para **2 samplers** (`TextureDiffuse` na unidade 0 + `TextureAux` na unidade 1 para especular/metalness/emissivo).
- `main.cpp`: as texturas agora são vinculadas **por objeto** (nova `BindTexturesForObject`, chamada pela `SetObjectId` que substituiu os `glUniform1i(g_object_id_uniform, …)`). `LoadTextureImage` guarda os IDs em vez de ocupar unidades fixas.

**3. Segfault na SoLoud** — o miniaudio 0.9.10 carrega os frameworks de áudio via `dlopen` com caminho relativo, o que falha em Apple Silicon (`MA_NO_BACKEND`). O `init()` falhava silenciosamente e o `play()` seguinte derreferenciava ponteiro nulo.
- Makefiles: `-DMA_NO_RUNTIME_LINKING` + `-framework CoreFoundation` → o miniaudio passa a usar os símbolos linkados diretamente. Isolei e confirmei: `init result = 0, backend = MiniAudio`.
