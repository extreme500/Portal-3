## Plano de Implementação: Colisão Jogador–Cenário (Cilindro + Slide)

### Achado crítico antes de tudo

As paredes/chão/teto/portas são todas instâncias de **um único mesh compartilhado** (`"the_plane"`, `"door_wall"`), desenhadas com `model = Matrix_Translate(...) * Matrix_Rotate_*(...) * Matrix_Scale(...)`. Logo `g_VirtualScene["the_plane"].bbox_min/max` é a AABB **em espaço local do objeto**, igual para todas as ~24 instâncias. **Não dá para usar essa AABB diretamente em espaço de mundo.** Duas opções:

- (A) Recalcular a AABB de mundo aplicando a `model` matrix de cada instância sobre os 8 cantos da AABB local, ou
- (B) Já que as salas têm geometria simples e **o usuário já descreveu as dimensões exatas das salas** (caixas/planos retos), **definir manualmente uma lista de AABBs de mundo** para paredes/chão/teto — muito mais simples, robusto e sem necessidade de tocar no pipeline de renderização.

Recomendo **(B)** para paredes/chão/teto/portas (geometria axis-aligned conhecida), e usar a AABB local de `g_VirtualScene[...]` transformada pela `model` apenas para objetos com geometria própria não-trivial como `"Cube"` (caixa) e `"portal_button_reduced_2"` (botão), aplicando (A) só a esses poucos.

Também confirmado: **não há gravidade nem pulo** (`SPACE` apenas reseta ângulos de Euler de depuração; `Pos_Player.y` nunca é alterado por input). Logo colisão vertical pode ser tratada como **clamp simples de altura** (chão/teto fixos por sala), sem física vertical de verdade.

### 2. Algoritmo cilindro vs AABB (simplificação círculo-XZ + faixa Y)

Como o cilindro é vertical e os obstáculos são AABBs (também alinhados aos eixos), a checagem 3D se decompõe em:

1. **Overlap vertical (eixo Y)**: faixa `[Pos_Player.y, Pos_Player.y + altura]` intersecta `[aabb.min.y, aabb.max.y]`?
2. **Overlap no plano XZ**: distância do **centro do círculo** (`Pos_Player.x, Pos_Player.z`) ao **retângulo** `[aabb.min.xz, aabb.max.xz]` é menor que `raio`? (clamp do centro ao retângulo + distância euclidiana — é exatamente o teste círculo-vs-AABB2D padrão.)

Função sugerida (inserir nas funções utilitárias, perto de `BezierCubic` ou junto das outras funções auxiliares de matemática, por volta da linha ~1100-1140, ou criar uma seção nova "Funções de colisão" antes de `main`):

```cpp
// Testa colisão entre o cilindro do jogador (centrado em pos_xz, com raio e
// faixa vertical [y_min, y_max]) e uma AABB de mundo. Retorna true se há sobreposição.
bool CylinderIntersectsAABB(const glm::vec2& pos_xz, float raio,
                            float y_min, float y_max,
                            const CollisionAABB& box)
{
    // 1) Overlap vertical
    if (y_max < box.min.y || y_min > box.max.y)
        return false;

    // 2) Distância do centro (XZ) ao retângulo da AABB projetada em XZ
    float closest_x = glm::clamp(pos_xz.x, box.min.x, box.max.x);
    float closest_z = glm::clamp(pos_xz.y, box.min.z, box.max.z); // pos_xz.y == coordenada Z
    float dx = pos_xz.x - closest_x;
    float dz = pos_xz.y - closest_z;

    return (dx*dx + dz*dz) < (raio * raio);
}

// Testa se o cilindro do jogador, na posição candidata 'pos', colide com
// QUALQUER AABB do cenário registrada em g_CollisionAABBs.
bool PlayerCollidesAt(const glm::vec3& pos)
{
    glm::vec2 pos_xz(pos.x, pos.z);
    float y_min = pos.y;
    float y_max = pos.y + g_PlayerCollider.altura;

    for (const CollisionAABB& box : g_CollisionAABBs)
    {
        if (CylinderIntersectsAABB(pos_xz, g_PlayerCollider.raio, y_min, y_max, box))
            return true;
    }
    return false;
}
```

### 3. Slide along walls — testar por eixo

A ideia: calcular o deslocamento desejado `(dx, dz)` (resultado de WASD nesse frame), então testar e aplicar **X e Z separadamente**:

```cpp
// Tenta mover o jogador por (dx, 0, dz). Aplica cada componente
// independentemente, permitindo "deslizar" ao longo de paredes quando apenas
// um eixo causa colisão.
void TryMovePlayer(float dx, float dz)
{
    // Testa eixo X isoladamente
    glm::vec3 candidate_x = glm::vec3(Pos_Player.x + dx, Pos_Player.y, Pos_Player.z);
    if (!PlayerCollidesAt(candidate_x))
        Pos_Player.x = candidate_x.x;

    // Testa eixo Z isoladamente (a partir da posição já potencialmente atualizada em X)
    glm::vec3 candidate_z = glm::vec3(Pos_Player.x, Pos_Player.y, Pos_Player.z + dz);
    if (!PlayerCollidesAt(candidate_z))
        Pos_Player.z = candidate_z.z;
}
```

Esse padrão (mover X, testar, mover Z, testar) é exatamente o "slide": se o movimento bate de frente numa parede perpendicular a Z mas o jogador também está se movendo em X, a componente X continua sendo aplicada livremente — produzindo o deslizar suave ao longo da parede.

```cpp
// Calcula a AABB de mundo de um objeto, dada sua AABB local (de g_VirtualScene)
// e a matriz de modelagem usada para desenhá-lo. Transforma os 8 cantos e
// recomputa min/max — válido mesmo com rotações/escalas.
CollisionAABB ComputeWorldAABB(const glm::vec3& local_min, const glm::vec3& local_max,
                               const glm::mat4& model)
{
    glm::vec3 corners[8] = {
        {local_min.x, local_min.y, local_min.z}, {local_max.x, local_min.y, local_min.z},
        {local_min.x, local_max.y, local_min.z}, {local_max.x, local_max.y, local_min.z},
        {local_min.x, local_min.y, local_max.z}, {local_max.x, local_min.y, local_max.z},
        {local_min.x, local_max.y, local_max.z}, {local_max.x, local_max.y, local_max.z},
    };

    glm::vec3 wmin( std::numeric_limits<float>::max());
    glm::vec3 wmax(-std::numeric_limits<float>::max());
    for (const glm::vec3& c : corners)
    {
        glm::vec3 wc = glm::vec3(model * glm::vec4(c, 1.0f));
        wmin = glm::min(wmin, wc);
        wmax = glm::max(wmax, wc);
    }
    return { wmin, wmax };
}
```

Total de objetos de colisão: ~12 paredes/chão/teto (6 por sala) + 2 estáticos (cubo, botão) = **~14 AABBs**. Um teste cilindro-AABB é O(1) com poucas operações aritméticas; 14 testes por eixo, 2 eixos, 60+ FPS — totalmente trivial (< 1 µs de CPU). **Força bruta (laço linear sobre `g_CollisionAABBs`) é mais que suficiente**; não há necessidade de grid, BVH, octree, ou qualquer estrutura espacial. Evitar essa complexidade adicional é a escolha correta para este escopo.
