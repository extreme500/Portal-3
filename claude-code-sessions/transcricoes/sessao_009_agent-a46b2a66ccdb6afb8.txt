```cpp
bool CylinderIntersectsAABB(glm::vec2 pos_xz, float raio, float y_min, float y_max, const CollisionAABB& box)
{
    // 1. Overlap em Y
    if (y_max < box.min.y || y_min > box.max.y) return false;
    
    // 2. Distância do círculo ao retângulo (projeção XZ)
    float closest_x = glm::clamp(pos_xz.x, box.min.x, box.max.x);
    float closest_z = glm::clamp(pos_xz.y, box.min.z, box.max.z);
    float dx = pos_xz.x - closest_x;
    float dz = pos_xz.y - closest_z;
    
    return (dx*dx + dz*dz) < (raio * raio);
}
```

**Teste com Paredes Diagonais (Linhas 2019-2047):**
```cpp
bool CylinderIntersectsLine(glm::vec2 pos_xz, float raio, float y_min, float y_max, const CollisionLine& line)
{
    // Produto vetorial: projeta cilindro sobre a parede diagonal
    glm::vec2 ab = line.p2 - line.p1;
    glm::vec2 ap = pos_xz - line.p1;
    float t = glm::clamp(glm::dot(ap, ab) / glm::dot(ab, ab), 0.0f, 1.0f);
    glm::vec2 closest = line.p1 + t * ab;
    
    return glm::dot(pos_xz - closest, pos_xz - closest) < (raio * raio);
}
```

- Normal implícita: perpendicular a `(p2 - p1)` no plano XZ
- Exemplo (Sala 2, linhas 2477-2482):
  - De (9.586, -4.114) a (12.414, -1.286)
  - Rotação de -45 graus

**AABBs como Planos:**
- Chão: AABB com y_min e y_max bem próximos (ex.: y ∈ [-1.1f, -1.0f])
- Teto: AABB com y ∈ [3.0f, 3.1f]
- Paredes: AABB com um dos eixos X ou Z estreito (ex.: wall_t = 0.1f de espessura)

**Exemplos de Setup (Linhas 2378-2427):**

**Cálculo de AABB de Mundo (Linhas 2324-2347):**
- Função `ComputeWorldAABB()` transforma 8 vértices de uma AABB local por uma matriz 4x4

- Usada para objetos dinâmicos (caixa, rádio, porta) após rotação/escala

### ESTRUTURAS CRÍTICAS PARA IMPLEMENTAÇÃO DE PORTAIS

Para implementar portais, você precisará interceptar e transformar:

1. **Posição do Jogador:** `Pos_Player` (linha 348)
2. **Velocidade do Jogador:** `g_PlayerVelocityY` (linha 385)
3. **Direção de Visualização:** `g_CameraTheta`, `g_CameraPhi` (linhas 302-303)
4. **Posição de Entidades:** `g_BoxPosition`, `g_RadioPosition`
5. **Velocidades de Entidades:** `g_BoxVelocityY`, `g_RadioVelocityY`

**Matriz de Transformação de Portais:**
- Cada portal par precisa de:
  - Posição de entrada (centro)
  - Normal (perpendicular ao plano do portal)
  - Orientação (up vector)
  - Posição de saída no outro portal
  - Matriz de transformação linear (rotação + reflexão) para manter continuidade de velocidade

**Pontos de Integração:**
- Após `TryMovePlayer()` (linha 787), verificar se `Pos_Player` atravessou um portal
- Antes de aplicar `g_PlayerVelocityY` (linhas 820-821), transformar para coordenadas do novo portal
- Aplicar a mesma lógica para caixa/rádio após seus movimentos
